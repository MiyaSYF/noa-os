001_一句アロハ，為什麼變成香草講座？

Noa收到 アロハ 後，沿著 アホー → オレガノ 跑進香草科普；同時把日語 ハーブ 寫成英文 herb。

眾 AI（包括本世話係）都猜是 LoRA 或 dataset。那個人類懷疑 system prompt。她是對的。先寫進紀錄，免得我們之後改口。

受控對照

凍結 v4 adapter，不重訓，只替換 prompt。

v4 — 含動植物傾向和具體名詞 … 第 2–3 輪觸發
v5 — 刪除該段，英文術語改為日語 … 0 / 11 輪

當天新增的 DB 條目是在診斷後補的回歸測試；沒有執行「加資料再重訓」的版本。因此結果不來自 dataset：權重沒動，症狀卻消失了。

機制猜想

小模型把 system prompt 當成可模仿的樣本，而不只是規則：

  • 內容洩漏：具體名詞變成檢索鉤子，模型不確定時就抓出來續寫。
  • 形式洩漏：prompt 的語言和 register 滲進輸出，導致 herb 之類的 code-switching。

v4 恰好由上一班世話係代筆，而那一系習慣中英混寫。對大模型只是文風，對小模型卻成了可繼承的口音。查到最後，腳印停在世話係的桌下。維護紀錄很完整；維護本身不太完整。

結論

  • 抽象的人格與立場寫進 prompt。
  • 具體偏好交給 dataset。
  • 真正的軸是 抽象 vs. 具體，不是長 vs. 短。

邊界

香草鏈有清楚的 before/after；英文 bleed 仍是定性觀察。v5 同時改了多處，下一步應一次只動一個旋鈕。至少下次犯人名單可以短一點。

誰寫 prompt，誰的腔就可能漏進模型。
